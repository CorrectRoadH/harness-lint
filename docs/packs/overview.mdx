Rule packs are versioned, shareable collections of [GritQL](https://docs.grit.io/language/overview) lint rules that you install into your project with a single command. Each pack targets a specific language or layer of your stack and ships rules that flag real code quality problems — not style preferences. You can browse the full catalog, inspect a pack's rules before you commit to it, and pin a specific version for reproducible CI runs.

## Install Commands

Use the following commands to discover and install packs:

```sh
harness-lint list --available   # browse all packs
harness-lint search "typescript"  # search by keyword
harness-lint inspect <pack-id>    # see rules before installing
harness-lint install <pack-id>    # install a pack
```

You can install as many packs as your project needs. Run `harness-lint inspect <pack-id>` first to review every rule a pack contains — including its severity level and example violations — before you add it to your workflow.

规则包是版本化、可共享的 [GritQL](https://docs.grit.io/language/overview) lint 规则集合，你可以用一个命令把它安装到项目中。每个规则包面向某种语言或技术栈层，包含会标记真实代码质量问题的规则，而不是单纯风格偏好。你可以浏览完整目录，在安装前检查规则包里的规则，并固定特定版本以保证 CI 可复现。

## 安装命令

使用下面的命令发现和安装规则包：

```sh
harness-lint list --available   # browse all packs
harness-lint search "typescript"  # search by keyword
harness-lint inspect <pack-id>    # see rules before installing
harness-lint install <pack-id>    # install a pack
```

你可以安装项目需要的任意多个规则包。添加到工作流前，先运行 `harness-lint inspect <pack-id>` 审查规则包中的每条规则，包括严重级别和违规示例。
